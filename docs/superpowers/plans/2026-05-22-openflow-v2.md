# OpenFlow V2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship V2 — bulk-migrate the remaining ~12 OpenTAP EVT tests, add the migrator transformers needed to handle whatever new patterns the bulk migration surfaces, and ship an HTML report renderer alongside the existing JSON. By the end of V2 every EVT test has a runnable pytest equivalent in `tests/` and engineers read a single-file `report.html` instead of raw JSON.

**Architecture:** New `openflow/report/` subpackage with a Jinja2-templated single-file HTML renderer that consumes the existing `report.json` schema unchanged. Two new migrator transformers (`RewriteSweepLoops`, `RewritePrintSummary`) close the last two automated cleanup steps; further transformers added on-demand as bulk migration surfaces patterns. CMW100 measurement surface grows on demand (LTE Tx ACLR/SEM, NR FFT/SA peak, etc.) — each new method ships with a unit test and an `is_emulation` path. Phase split: V2a (framework + transformers + renderer) is hardware-independent; V2b (bench validation of 2 representative tests) needs the bench engineer.

**Tech Stack:** Python 3.11 · uv · pytest · libcst (transformer additions) · jinja2 (HTML rendering) · existing R&S SDK packages · ruff · mypy · GitHub Actions

Spec: `docs/superpowers/specs/2026-05-22-openflow-v2-design.md`

```python
class RewriteSweepLoops(cst.CSTTransformer):

    - its body contains no calls to methods on the test's fixture-bound
      objects (dut, cmw100, wfg, …) — those are setup side effects
    - it is the outermost statement in the test function body
    """
```

# Stop conditions

Pause V2 (re-scope before continuing) if any of the following hit:

- **Phase 1 inventory finds >6 new patterns.** That's a signal the migrator isn't generalizing — better to slow down and design rather than power through.
- **HTML rendering balloons past 200 KB per test session.** Means we're including too much; reduce inlined assets.
- **More than 25 new CMW100 methods needed.** That implies a fundamentally different test surface; might be worth waiting for the bench engineer to triage.
- **No engineer EVT source available.** V2a can ship without the new tests (synthetic fixtures + canonical fixture only) but V2 success criterion #1 is unmet.
